# GOP 탐지 스냅샷 — AWS S3 연동 가이드

## 0. 개념: S3 vs RDS

| | RDS (MySQL) | S3 |
|---|---|---|
| 저장 대상 | 정형 데이터 (행/열) | 파일 (이미지·영상) |
| 비유 | 엑셀 스프레드시트 | 구글 드라이브 |
| 이번 용도 | 탐지 시각·신뢰도·상태 | 탐지 순간 캡처 이미지 |

이미지를 DB에 직접 넣는 것은 안티패턴입니다. **이미지는 S3에, 그 경로는 RDS의 `image_path` 컬럼에** 저장합니다.

## 1. 사용할 파일 구성

```
프로젝트 폴더/
├── frontend_multi_rds_s3_ver1.py   ← S3 연동 추가된 프론트엔드
├── db_rds.py                    ← 기존 그대로
├── s3_storage.py                ← 신규: S3 업로드/조회/삭제 모듈
├── backend.py                   ← 기존 그대로
└── .streamlit/
    └── secrets.toml             ← [s3] 섹션 추가
```

## 3. S3 버킷 만들기 (콘솔)

1. AWS 콘솔 → 검색창 **S3** → 진입.
2. **버킷 만들기** 클릭.
3. 버킷 이름: 예) `gop-detection-images` (전 세계에서 고유해야 함).
4. AWS 리전: **아시아 태평양(서울) ap-northeast-2**.
5. 객체 소유권: 기본값(ACL 비활성화) 유지.
6. **퍼블릭 액세스 차단**: **모두 차단(기본값) 유지** ← 중요.
   → 이미지는 presigned URL로만 접근하므로 버킷을 공개할 필요가 없습니다.
7. 나머지 기본값 유지 → **버킷 만들기**.

---

## 4. IAM 사용자 만들기 (액세스 키 발급)

S3에 업로드하려면 권한을 가진 자격증명이 필요합니다.

1. AWS 콘솔 → **IAM** → **사용자** → **사용자 생성**.
2. 사용자 이름: 예) `gop-s3-uploader`.
3. 권한 설정 → **직접 정책 연결** → `AmazonS3FullAccess` 검색 후 체크.
   → (운영 시에는 해당 버킷에만 권한을 주는 커스텀 정책을 권장하지만, 데모는 이걸로 충분)
4. 사용자 생성 완료 후, 해당 사용자 → **보안 자격 증명** 탭 → **액세스 키 만들기**.
5. 사용 사례: **로컬 코드(Local code)** 선택 → 생성.
6. **액세스 키 ID**와 **비밀 액세스 키**를 복사해 둡니다.
   → 비밀 키는 이 화면을 벗어나면 다시 볼 수 없으니 반드시 저장하세요.

## 5. secrets.toml 설정

`.streamlit/secrets.toml`에 기존 DB 설정 아래로 `[s3]` 섹션을 추가합니다.

```toml
[s3]
bucket            = "gop-detection-images"
region            = "ap-northeast-2"
access_key_id     = "AKIA..."
secret_access_key = "..."
```

`[s3]` 섹션이 없으면 S3 기능은 자동으로 비활성화되고, 기존처럼 메모리 스냅샷만 사용합니다(앱은 정상 작동).

## 6. 동작 방식

```
[탐지 발생]
  → s3.upload_snapshot()  : 이미지를 S3에 JPEG로 업로드
      키 형식: detections/2026-06-23/CCTV-01_a1b2c3d4.jpg
  → 반환된 키를 record["image_path"]에 저장
  → db.insert_log()       : 메타데이터 + image_path를 RDS에 기록

[화면 크게 보기 — 재시작 후]
  → 메모리 스냅샷 없음
  → image_path(S3 키)로 presigned URL 발급
  → 이미지 표시 (앱을 껐다 켜도 복원됨)

[로그 삭제]
  → RDS 행 삭제 + S3 이미지 함께 삭제 (고아 파일 방지)
```

### 키 포인트
- 이번 세션 탐지: **메모리 스냅샷** 우선 사용 (빠름)
- 재시작 후 복원된 로그: **S3 presigned URL**로 이미지 표시
- presigned URL은 비공개 버킷에서도 임시 접근(기본 1시간)을 허용

---

## 7. 동작 확인

1. 앱 실행 후 데모 모드로 사람 탐지를 발생시킵니다.
2. S3 콘솔 → 버킷 → `detections/날짜/` 폴더에 `.jpg`가 쌓이는지 확인.
3. **앱을 껐다 켠 뒤** 탐지 로그에서 해당 항목의 "화면 크게"를 눌러
   이미지가 복원되면 성공입니다.

### 문제 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 사이드바에 S3 경고 | 자격증명 오타 | secrets.toml의 access_key 확인 |
| `AccessDenied` | IAM 권한 부족 | `AmazonS3FullAccess` 연결 확인 |
| `NoSuchBucket` | 버킷 이름 불일치 | bucket 값과 실제 버킷명 일치 확인 |
| 이미지가 안 보임 | 리전 불일치 | region이 버킷 생성 리전과 같은지 확인 |

## 8. 비용 참고

S3 프리 티어(12개월): 5GB 저장 + 월 2만 건 PUT + 월 20만 건 GET 무료.
탐지 이미지 1장이 평균 100KB라면 5GB로 약 5만 장 저장 가능 — 데모/포트폴리오 기간에는 비용 걱정이 없습니다.
